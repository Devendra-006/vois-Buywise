## AI Contract Analyzer – Technical Stack (Agentic System)

This document outlines the recommended technical stack to implement the **AI Contract Analyzer with multi‑agent architecture**.

---

## 1. Core Platform Architecture

- **Backend Framework**
  - Python `3.11+`
  - `FastAPI` (primary HTTP API, async, OpenAPI docs)
  - `Uvicorn` / `Gunicorn` (ASGI server)

- **Frontend**
  - `React` or `Next.js` (SPA/SSR for dashboard, upload flow, chat UI)
  - UI toolkit: `MUI` or `Tailwind CSS` (for fast, modern UI)
  - Charting: `Recharts` or `Chart.js` (risk dashboards and scores)

- **Database**
  - `PostgreSQL` (primary relational database for users, contracts, runs, settings)
  - ORM: `SQLAlchemy` or `Prisma` (if using TypeScript backend in the future)

---

## 2. AI, NLP & Agentic System

- **LLM Access Layer**
  - OpenAI / Azure OpenAI / Anthropic / Gemini (any production LLM provider)
  - Python SDKs (e.g. `openai`, `anthropic`, `google-generativeai`) or OpenAI-compatible HTTP clients

- **Agent Orchestration (Multi‑Agent System)**
  - `LangGraph` or `LangChain` (graph-based multi‑agent workflows)
  - Agent roles:
    - Document Processing Agent
    - Clause Detection Agent
    - Law Retrieval Agent (RAG)
    - Risk Analysis Agent
    - Risk Score Agent
    - Suggestion / Negotiation Agent
    - Summary Agent
    - (Optional) Contract Chat Agent & Comparison Agent

- **NLP & Clause Identification**
  - `spaCy` or `transformers` (NER, clause classification)
  - Pretrained legal / contract models if available (e.g. `legal-bert`-style models)
  - Custom prompt templates for clause extraction using LLMs

---

## 3. Document Ingestion & OCR

- **File Handling**
  - Supported formats: PDF, DOCX, image scans (JPEG/PNG)
  - Libraries:
    - `pypdf` or `pdfplumber` (text extraction from PDFs)
    - `python-docx` (DOCX parsing)
    - `fastapi-upload-file` / `aiofiles` (async file upload handling)

- **OCR for Scanned Contracts**
  - `Tesseract OCR` with `pytesseract` (Python binding)
  - Image preprocessing: `Pillow` / `OpenCV` (deskew, binarization, resize)

---

## 4. Retrieval-Augmented Generation (RAG) for Legal Laws

- **Vector Database**
  - Recommended options (choose one):
    - `Chroma` (embedded, simple to start)
    - `FAISS` (local vector index)
    - `Pinecone` (managed cloud vector DB)

- **Embedding Models**
  - OpenAI text embeddings or local embedding model via `sentence-transformers`

- **Legal Corpus Ingestion**
  - Scraping / ingestion scripts for:
    - India Code
    - Ministry of Law and Justice
    - Relevant acts (Indian Contract Act 1872, IT Act 2000, Consumer Protection Act 2019, etc.)

---

## 5. Risk Analysis, Scoring & Dashboard

- **Risk Engine**
  - Rule-based + LLM‑assisted scoring:
    - Python modules encapsulating scoring logic per clause type
    - Configuration for thresholds (e.g., penalty months, duration, liability caps)

- **Analytics & Visualization**
  - Expose scoring breakdown via FastAPI endpoints
  - Frontend charts: radar/bar charts for clause‑wise scores

---

## 6. Infrastructure, DevOps & Observability

- **Environment & Packaging**
  - `poetry` or `pip` + `requirements.txt` for Python dependency management
  - `.env` configuration using `python-dotenv` / environment variables

- **Serving & Deployment**
  - Containerization with `Docker`
  - Reverse proxy: `NGINX` or `Traefik`
  - Deployment targets: `AWS` / `GCP` / `Azure` / `Railway` / `Render`

- **Monitoring & Logging**
  - Structured logging with `loguru` or Python `logging`
  - Request tracing, latency & error metrics (e.g. `Prometheus` + `Grafana`, or cloud-native tools)

---

## 7. Security, Compliance & Access Control

- **Authentication & Authorization**
  - JWT-based auth (e.g. `fastapi-users` or custom)
  - Role-based access (user, admin)

- **Security Practices**
  - HTTPS/TLS termination at proxy
  - Secure file storage (contracts are sensitive)
  - Basic PII handling & data retention policies

---

## 8. Project Tooling & Quality

- **Testing**
  - `pytest` for backend
  - `jest` / `vitest` + `React Testing Library` for frontend

- **Code Quality**
  - Linters/formatters: `ruff`, `black`, `isort` (Python), `eslint`, `prettier` (JS/TS)

- **Documentation**
  - API docs via FastAPI’s autogenerated OpenAPI
  - End‑user documentation in `README.md` and this `tech-stack.md`
